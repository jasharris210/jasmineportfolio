Bug Bounty Program

Duke OIT | 05.2020 - 07.2020

Role: UX Designer/ Front-end developer

Software/languages used: HTML, CSS, Python, aerosprite, javascript

The Problem:

As websites worldwide continue to have problems with hackers and their malicious attacks, there is a high demand to improve cybersecurity. The Duke office of indormation< technology wanted to protect duke domains by reducing these attacks.

The Solution:

Creating a bug bounty program : an incentive-based initiative that rewards ethical hackers for submitting vulnerabilities, or bugs, that we fix. This platform provides students with hands-on experience in cybersecurity while also improving the security of Duke systems.

Wireframe
Using lucid chart we created a wireframe for out program. To attract more users especially students we decided to make the user interface have a video-game aesthetic.

The beginning of the BB program. Once you "press any button" and fill in your information for the Oauth login, will take you to the admin or user page, depending on your credentials.

The user pages include a submission page, where users will submit the bug that they have found, a how-to-play page, which has a turotial on how to use the BB program an FAQ, and more information about the program, and the rewards page, which will inform users how much money they can earn.

These are the admin pages that will allow admins to filter out the eligible bugs and grant users the correct reward.

User profiles that will include more information about the user.

Uniqueness of Duke
Bug Bounty

We put an emphasis on a user-friendly program that makes bug bounty hunting much more approachable and comfortable even for first time users. We did this in three major ways:

Simplify
All technical terminology and jargon will be explained in the vernacular.

Illustrate
create a video tutorial that shows students exactly how to play and explains the rules in clear detail.

Gamify
Idea is to make it fun and engaging and introduce element of competitiveness.

Results
we had successfully created the bug bounty program where all Duke community members can report security vulnerabilities. We had connected these submissions to a ServiceNow API, where OIT reviews and resolves them. Finally, We implemented the admin pages where the OIT would evaluate each resolved ticket and grant their award depending on the bug that they had submitted.
